# Task: Alembic env.py + alembic.ini.template

## Description

Create the Alembic migration environment. Because all models are abstract, `alembic/versions/` ships empty — agents generate their own initial migration after subclassing.

Files to create:
- `TEMPLATE_CORE/USER/alembic/env.py`
- `TEMPLATE_CORE/USER/alembic.ini.template`

## Acceptance Criteria
- [ ] `env.py` imports `Base` from `user_core.db`
- [ ] `env.py` has a comment instructing agents to import their concrete models before `target_metadata = Base.metadata` so autogenerate detects their tables
- [ ] `env.py` reads `DATABASE_URL` from environment via `python-dotenv`
- [ ] `env.py` implements both `run_migrations_offline()` and `run_migrations_online()`
- [ ] `alembic.ini.template` uses `sqlalchemy.url = %(DATABASE_URL)s` pattern
- [ ] `alembic/versions/` directory exists (empty — no initial migration shipped)

## Technical Details

`env.py` comment block for agents:
```python
# AGENT SETUP: Import your concrete models here so Alembic can detect them.
# Example:
#   from myapp.models import User, Order  # noqa: F401
import user_core.models  # noqa: F401 — imports abstract classes; registers nothing
```

Note: importing `user_core.models` is sufficient to ensure `Base` is initialized, but since all models are abstract, `Base.metadata.tables` remains empty. Agents must import THEIR concrete models to populate metadata.
