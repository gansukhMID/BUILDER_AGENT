# Task: PickingType Model

## Description

Implement the `PickingType` model — the operation type definition that configures how transfers behave (receipt, delivery, internal transfer).

Files to create:
- TEMPLATE_CORE/WMS/wms_core/models/picking_type.py

PickingType model fields:
- `id: Mapped[int]` (primary key)
- `warehouse_id: Mapped[Optional[int]]` (FK to Warehouse, nullable — some types are cross-warehouse)
- `code: Mapped[str]` (enum: "incoming", "outgoing", "internal"; non-nullable)
- `default_location_src_id: Mapped[Optional[int]]` (FK to Location, nullable)
- `default_location_dest_id: Mapped[Optional[int]]` (FK to Location, nullable)
- `sequence_prefix: Mapped[Optional[str]]` (e.g., "IN/", "OUT/", nullable)
- `count_picking_ready: Mapped[int]` (computed-style counter, default 0 — updated by app logic)
- Inherits: `TimestampMixin`, `ActiveMixin`, `NameMixin`
- Relationships: `warehouse`, `default_src_location`, `default_dest_location`

## Acceptance Criteria

Two FKs to Location table on the same model — SQLAlchemy requires explicit `foreign_keys=[...]` on each relationship. Use string references for FKs to avoid circular imports.
