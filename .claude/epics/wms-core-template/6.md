# Task: Picking Model + StateMachine Utility

## Description

Implement `utils/state_machine.py` generic FSM helper and the `Picking` model with a full transfer state machine.

Files to create:
- TEMPLATE_CORE/WMS/wms_core/utils/state_machine.py
- TEMPLATE_CORE/WMS/wms_core/models/picking.py

state_machine.py:
- `class InvalidTransition(Exception)` — raised when transition not allowed
- `class StateMachine`: initialized with `transitions: dict[str, list[str]]` (state → allowed next states)
  - `can(current_state: str, target_state: str) -> bool`
  - `apply(obj: Any, state_attr: str, target_state: str) -> None` — sets obj.state_attr if transition valid, raises InvalidTransition otherwise

Picking model fields:
- `id: Mapped[int]` (primary key)
- `picking_type_id: Mapped[int]` (FK to PickingType, non-nullable)
- `partner_id: Mapped[Optional[int]]` (nullable int — agent adds FK to Partner model)
- `location_src_id: Mapped[int]` (FK to Location)
- `location_dest_id: Mapped[int]` (FK to Location)
- `state: Mapped[str]` (default "draft"; values: draft, confirmed, in_progress, done, cancelled)
- `scheduled_date: Mapped[Optional[datetime]]` (nullable)
- `origin: Mapped[Optional[str]]` (source document reference, nullable)
- Inherits: `TimestampMixin`, `NameMixin`
- Relationships: `picking_type`, `src_location`, `dest_location`, `moves` (list of Move, back_populates)
- FSM instance (class-level): `_fsm = StateMachine({"draft": ["confirmed", "cancelled"], "confirmed": ["in_progress", "cancelled"], "in_progress": ["done", "cancelled"], "done": [], "cancelled": []})`
- Methods:
  - `confirm() -> None` — transitions draft → confirmed
  - `start() -> None` — transitions confirmed → in_progress
  - `validate() -> None` — stub: transitions in_progress → done (subclasses override to move quants)
  - `cancel() -> None` — transitions any non-done state → cancelled
