Move model fields:
- `id: Mapped[int]` (primary key)
- `picking_id: Mapped[Optional[int]]` (FK to Picking, nullable — moves can exist without picking)
- `product_id: Mapped[int]` (FK to Product)
- `location_src_id: Mapped[int]` (FK to Location)
- `location_dest_id: Mapped[int]` (FK to Location)
- `product_qty: Mapped[float]` (planned quantity)
- `qty_done: Mapped[float]` (default 0.0)
- `lot_id: Mapped[Optional[int]]` (FK to Lot, nullable)
- `state: Mapped[str]` (default "draft"; mirrors Picking states)
- Inherits: `TimestampMixin`
- Method: `validate(session) -> None` — sets state="done", calls `Quant.add_quantity` for src (negative) and dest (positive)
- Relationships: `picking`, `product`, `src_location`, `dest_location`, `lot`

StockRule model fields:
- `id: Mapped[int]` (primary key)
- `picking_type_id: Mapped[int]` (FK to PickingType)
- `location_src_id: Mapped[Optional[int]]` (FK to Location, nullable)
- `location_dest_id: Mapped[int]` (FK to Location)
- `warehouse_id: Mapped[Optional[int]]` (FK to Warehouse, nullable)
- `action: Mapped[str]` (enum: "pull", "push", "pull_push"; default "pull")
- `procure_method: Mapped[str]` (enum: "make_to_stock", "make_to_order"; default "make_to_stock")
- `auto: Mapped[str]` (enum: "manual", "transparent"; default "manual")
- Inherits: `TimestampMixin`, `ActiveMixin`, `NameMixin`

Package model (stub):
- `id: Mapped[int]` (primary key)
- `name: Mapped[str]` (non-nullable)
- `location_id: Mapped[Optional[int]]` (FK to Location, nullable)
- `__tablename__` = `"stock_quant_package"`

Alembic:
- alembic/env.py: imports Base from wms_core.db, reads DATABASE_URL from env (python-dotenv), uses `target_metadata = Base.metadata`
- alembic.ini.template: script_location = alembic, sqlalchemy.url = %(DATABASE_URL)s
- alembic/versions/0001_initial.py: auto-generated migration (run `alembic revision --autogenerate -m "initial"` after all models are imported in env.py); commit the output

Test suite coverage targets (≥80% on wms_core/):
- test_models.py: instantiate all models, verify __tablename__, verify relationships load
- test_quant.py: Quant.add_quantity round trip (call twice → single quant with sum); Quant.get_available; receipt → delivery cycle (qty goes up then down)
- test_state_machine.py: all valid transitions; InvalidTransition on invalid; Picking.validate() raises from wrong state
- test_hierarchy.py: complete_name for 3-level hierarchy; is_ancestor_of; get_all_children returns all descendants

For the test suite, use an in-memory SQLite engine (`sqlite:///:memory:`) with `Base.metadata.create_all(engine)` in a pytest fixture. Alembic env.py must import all models (via `from wms_core.models import *`) before `target_metadata = Base.metadata` so autogenerate detects all tables.
