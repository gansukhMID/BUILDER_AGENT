# Task: Product Model

## Description

Implement the `Product` model — the storable item definition with UoM and lot tracking configuration.

Files to create:
- TEMPLATE_CORE/WMS/wms_core/models/product.py

Product model fields:
- `id: Mapped[int]` (primary key)
- `uom: Mapped[str]` (unit of measure as string enum: "unit", "kg", "liter", "meter", "box"; default "unit")
- `tracking: Mapped[str]` (enum: "none", "lot", "serial"; default "none")
- `can_be_sold: Mapped[bool]` (default True)
- `can_be_purchased: Mapped[bool]` (default True)
- `sale_price: Mapped[Optional[float]]` (nullable)
- `cost_price: Mapped[Optional[float]]` (nullable)
- Inherits: `TimestampMixin`, `ActiveMixin`, `NameMixin`

## Acceptance Criteria
- [ ] `from wms_core.models.product import Product` succeeds
- [ ] `Product.__tablename__` = `"product_product"`
- [ ] `tracking` defaults to `"none"`
- [ ] `uom` defaults to `"unit"`
- [ ] `can_be_sold` and `can_be_purchased` default to `True`

UoM is stored as a string, not a relational model — keep the template lean. No external product.template / product.category split (single-level, Odoo-simplified). `NameMixin` provides `name` and `code` (internal reference).
