# Task: Quant Model

## Description

Implement the `Quant` model — the materialized on-hand inventory table, the single source of truth for current stock levels.

Files to create:
- TEMPLATE_CORE/WMS/wms_core/models/quant.py

Quant model fields:
- `id: Mapped[int]` (primary key)
- `product_id: Mapped[int]` (FK to Product, non-nullable)
- `location_id: Mapped[int]` (FK to Location, non-nullable)
- `lot_id: Mapped[Optional[int]]` (FK to Lot, nullable)
- `quantity: Mapped[float]` (on-hand quantity, can be negative for transit; default 0.0)
- `reserved_quantity: Mapped[float]` (reserved for outgoing moves; default 0.0)
- Inherits: `TimestampMixin` only
- Unique constraint: `(product_id, location_id, lot_id)` — one quant per (product, location, lot) combination
- Relationships: `product`, `location`, `lot`

Class methods:
- `add_quantity(cls, session, product_id, location_id, qty, lot_id=None) -> "Quant"` — find existing quant by (product, location, lot) and increment quantity, or create new with that quantity. Use session.merge or query+upsert pattern.
- `get_available(cls, session, product_id, location_id, lot_id=None) -> float` — returns `quantity - reserved_quantity` for the matching quant, or 0.0 if not found.

## Acceptance Criteria
- [ ] `from wms_core.models.quant import Quant` succeeds
- [ ] `Quant.__tablename__` = `"stock_quant"`
- [ ] Unique constraint on `(product_id, location_id, lot_id)` enforced
- [ ] `add_quantity` called twice with same (product, location, lot) results in single quant with summed quantity
- [ ] `get_available` returns `quantity - reserved_quantity`
- [ ] `get_available` returns 0.0 when no quant exists

## Technical Details

Unique constraint with nullable lot_id: use `UniqueConstraint("product_id", "location_id", "lot_id")` — SQLAlchemy handles NULL correctly on most DBs. `add_quantity` should query for existing quant first (select for update or plain select), then update or insert. Keep it simple — no SQLAlchemy merge magic needed.
