# Task: Warehouse Model

## Description

Implement the `Warehouse` model — the top-level facility that groups a set of stock locations.

Files to create:
- TEMPLATE_CORE/WMS/wms_core/models/warehouse.py

Warehouse model fields:
- `id: Mapped[int]` (primary key)
- `lot_stock_id: Mapped[int]` (FK to Location — main stock location)
- `wh_input_stock_loc_id: Mapped[Optional[int]]` (FK to Location — input gate, nullable)
- `wh_output_stock_loc_id: Mapped[Optional[int]]` (FK to Location — output gate, nullable)
- `reception_steps: Mapped[str]` (enum: "one_step", "two_steps", "three_steps"; default "one_step")
- `delivery_steps: Mapped[str]` (enum: "one_step", "two_steps", "three_steps"; default "one_step")
- Inherits: `TimestampMixin`, `ActiveMixin`, `NameMixin`
- Relationships: `lot_stock_location`, `input_location`, `output_location` → Location

## Acceptance Criteria
- [ ] `from wms_core.models.warehouse import Warehouse` succeeds
- [ ] `Warehouse.__tablename__` = `"stock_warehouse"`
- [ ] FK to Location resolves without circular import
- [ ] `reception_steps` defaults to `"one_step"`
- [ ] `Warehouse` can be instantiated with just `name` + `lot_stock_id`

## Technical Details

FKs to Location use string reference `"Location.id"` (or the table name `"stock_location.id"`) to avoid circular imports since warehouse.py imports from location.py. Use `foreign_keys=[...]` on relationships if SQLAlchemy needs disambiguation (multiple FKs to same table).
