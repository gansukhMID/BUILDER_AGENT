## Technical Details

Both classes in `order.py` — one file is fine since they're tightly coupled conceptually.

`line_total` is a stored column (not a computed/hybrid property) — the application layer must set it. This gives agents flexibility (line-level discounts, taxes, etc.).

`reference` is intentionally not unique in the abstract — agents decide if order references are globally unique or scoped (e.g., per-user).
