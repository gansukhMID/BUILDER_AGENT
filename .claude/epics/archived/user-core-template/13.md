# Task: Foundation — db.py, mixins.py, enums.py

## Description

Create the three foundation modules that all model files depend on. This task must complete before any model task can start.

Files to create:
- `TEMPLATE_CORE/USER/user_core/db.py`
- `TEMPLATE_CORE/USER/user_core/mixins.py`
- `TEMPLATE_CORE/USER/user_core/enums.py`

## Acceptance Criteria
- [ ] `db.py` exports `Base` (DeclarativeBase), `engine_from_url(dsn)`, `session_factory(engine)`, `get_session(engine)` context manager
- [ ] `Base.metadata.tables` is empty after import (abstract models register nothing)
- [ ] `mixins.py` exports `PrimaryKeyMixin` (`id: Mapped[int]`, primary_key), `TimestampMixin` (`created_at`, `updated_at` with `server_default=func.now()`), `SoftDeleteMixin` (`deleted_at: Mapped[datetime | None]`, `is_deleted` property)
- [ ] `enums.py` exports `MembershipTier` (free/pro/vip), `OrderState` (draft/confirmed/paid/shipped/delivered/cancelled), `PurchaseStatus` (pending/completed/refunded/failed), `ReviewStatus` (pending/approved/rejected)
- [ ] All enums are `str, enum.Enum` subclasses for SQLAlchemy compatibility

## Technical Details

`db.py` pattern (SQLAlchemy 2.x):
```python
from sqlalchemy.orm import DeclarativeBase
class Base(DeclarativeBase):
    pass
```

`PrimaryKeyMixin` goes into every abstract base — agents needing UUID PKs override `id` in their subclass.

`SoftDeleteMixin.is_deleted` is a Python property (`return self.deleted_at is not None`), not a DB column.

Enum naming convention: `name="membership_tier"`, `name="order_state"`, etc. — snake_case of the class name.
