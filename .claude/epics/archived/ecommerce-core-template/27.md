# Task: Product Catalog Models

## Description

Implement `ProductCategory`, `ProductTemplate`, `ProductVariant`, `ProductAttribute`, `ProductAttributeValue`, and the `ProductVariantAttributeLine` association table. These models form the core catalog layer that all commerce models reference.

## Acceptance Criteria
- [ ] `ecommerce_core/models/category.py` — `ProductCategory` with self-referential `parent_id`, `complete_name` property
- [ ] `ecommerce_core/models/product.py` — `ProductTemplate` (catalog unit) and `ProductVariant` (purchasable SKU)
- [ ] `ecommerce_core/models/attribute.py` — `ProductAttribute`, `ProductAttributeValue`, `ProductVariantAttributeLine` association table
- [ ] `ProductCategory.complete_name` returns full path string (e.g. `"Electronics / Phones / Accessories"`)
- [ ] `ProductVariant.display_name` returns `"<template name> — <val1> / <val2>"` from linked attribute values
- [ ] `ProductVariant.sku` has a unique constraint
- [ ] All models use `TimestampMixin` and `ActiveMixin` from `ecommerce_core.mixins`
- [ ] No circular imports — each model in its own file

## Technical Details

**ProductCategory** (`category.py`):
- Fields: `id`, `name`, `parent_id` (FK self), `active`
- Relationship: `parent`, `children`
- Property: `complete_name` — walks parent chain, joins with ` / `

**ProductTemplate** (`product.py`):
- Fields: `id`, `name`, `description`, `category_id` (FK `ProductCategory`), `list_price` (Numeric 12,4), `image_url`, `active`, `created_at`, `updated_at`
- Relationship: `category`, `variants`

**ProductVariant** (`product.py`):
- Fields: `id`, `template_id` (FK `ProductTemplate`), `sku` (unique), `barcode`, `default_price` (Numeric 12,4), `active`, `created_at`, `updated_at`
- Relationship: `template`, `attribute_lines`
- Property: `display_name`

**ProductAttribute** (`attribute.py`):
- Fields: `id`, `name`, `active`
- Relationship: `values`

**ProductAttributeValue** (`attribute.py`):
- Fields: `id`, `attribute_id` (FK), `name`, `active`

**ProductVariantAttributeLine** (`attribute.py`):
- Association table: `variant_id`, `attribute_value_id` (composite PK)
