# Task: Cart & Checkout

## Description

Implement `Cart` and `CartLine` models with full line management, total computation, and a `checkout()` stub. Cart is the shopping session layer; `checkout()` is the bridge to `SaleOrder` (implemented as a stub that agents complete once SaleOrder exists).

## Acceptance Criteria
- [ ] `ecommerce_core/models/cart.py` — `Cart` and `CartLine` models
- [ ] `Cart.partner_id` nullable (supports guest checkout)
- [ ] `Cart.session_token` is unique and indexed
- [ ] `Cart.add_line(variant, qty)` upserts a `CartLine` (adds qty if line exists)
- [ ] `Cart.remove_line(variant)` deletes the matching `CartLine` if present (no-op otherwise)
- [ ] `Cart.compute_totals()` returns dict: `{'subtotal', 'discount', 'tax_amount', 'shipping_estimate', 'total'}` all as `Decimal`
- [ ] `Cart.checkout(shipping_method_id, billing_address_id, shipping_address_id)` raises `NotImplementedError` with message explaining agents must implement after wiring `SaleOrder`
- [ ] `Cart.status` transitions: `active` → `abandoned` / `checked_out` (no FSM needed, direct assignment)
- [ ] Tax amount computed as sum of `CartLine.qty * variant.default_price * tax_rate` (use `0.0` default since tax is on order line — stub)
- [ ] Shipping estimate delegates to `ShippingMethod.price` if `cart.shipping_method_id` set, else `Decimal('0')`
- [ ] All models use `TimestampMixin`

## Technical Details

**Cart** (`cart.py`):
- Fields: `id`, `partner_id` (nullable FK `Partner`), `session_token` (unique, indexed), `pricelist_id` (nullable FK `Pricelist`), `coupon_id` (nullable FK `Coupon`), `shipping_method_id` (nullable FK `ShippingMethod`), `status` (Enum: `active`, `abandoned`, `checked_out`, default `active`), `created_at`, `updated_at`
- Relationships: `partner`, `pricelist`, `coupon`, `lines`, `shipping_method`
- Methods: `add_line()`, `remove_line()`, `compute_totals()`, `checkout()`

**CartLine** (`cart.py`):
- Fields: `id`, `cart_id` (FK `Cart`), `variant_id` (FK `ProductVariant`), `qty` (Numeric 10,4, default 1)
- Unique constraint: `(cart_id, variant_id)`
- Relationships: `cart`, `variant`
