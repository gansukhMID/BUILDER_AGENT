# Task: Pricing Models & Engine

## Description

Implement `Pricelist`, `PricelistItem`, and `utils/pricing.py` evaluation helpers. `Pricelist.get_price(variant, qty)` is the core method agents use to resolve the effective price at checkout.

## Acceptance Criteria
- [ ] `ecommerce_core/models/pricelist.py` — `Pricelist` and `PricelistItem` models
- [ ] `ecommerce_core/utils/pricing.py` — `evaluate_pricelist(pricelist, variant, qty)` pure function
- [ ] `Pricelist.get_price(variant, qty)` evaluates items in priority order and returns `Decimal` price
- [ ] Three `compute_price` modes supported: `fixed`, `percentage` (discount off `list_price`), `formula` (stub — raises `NotImplementedError` for agents to implement)
- [ ] `PricelistItem.applied_on` scopes: `all`, `category`, `product`, `variant`
- [ ] Items matched by specificity: `variant` > `product` > `category` > `all`; ties broken by `min_quantity` descending
- [ ] Falls back to `ProductVariant.default_price` if no pricelist item matches
- [ ] All models use `TimestampMixin`, `ActiveMixin`, `CurrencyMixin`

## Technical Details

**Pricelist** (`pricelist.py`):
- Fields: `id`, `name`, `currency` (default `'USD'`), `active`, `created_at`, `updated_at`
- Relationship: `items`
- Method: `get_price(variant: ProductVariant, qty: Decimal) -> Decimal`

**PricelistItem** (`pricelist.py`):
- Fields: `id`, `pricelist_id` (FK), `applied_on` (Enum: `all`, `category`, `product`, `variant`), `product_variant_id` (nullable FK), `product_template_id` (nullable FK), `category_id` (nullable FK), `compute_price` (Enum: `fixed`, `percentage`, `formula`), `fixed_price` (Numeric, nullable), `price_discount` (Numeric 5,2, nullable — percentage off), `min_quantity` (Numeric, default 0), `priority` (int, default 0)

**utils/pricing.py**:
- `evaluate_pricelist(pricelist, variant, qty) -> Decimal` — filters applicable items, sorts by specificity + priority + min_quantity, applies compute logic
