# Task: SaleOrder & SaleOrderLine

## Description

Implement `SaleOrder` and `SaleOrderLine` with a full state machine, line totals, coupon application, and shipping/billing address linkage. This is the central commerce record that `Cart.checkout()` creates.

## Acceptance Criteria
- [ ] `ecommerce_core/models/order.py` — `SaleOrder` and `SaleOrderLine` models
- [ ] `SaleOrder.state` state machine enforced via `utils/state_machine.py`:
  - Valid transitions: `draft→confirmed`, `confirmed→processing`, `processing→shipped`, `shipped→delivered`, `draft→cancelled`, `confirmed→cancelled`, `delivered→refunded`
  - Invalid transitions raise `ValueError` with clear message
- [ ] `SaleOrder.confirm()` transitions to `confirmed`; raises `ValueError` if no lines
- [ ] `SaleOrder.cancel()` transitions to `cancelled`; raises `ValueError` if state is `shipped`, `delivered`, or `refunded`
- [ ] `SaleOrder.mark_shipped(tracking_number)` transitions to `shipped`; stores `tracking_number`
- [ ] `SaleOrder.mark_delivered()` transitions to `delivered`
- [ ] `SaleOrder.apply_coupon(code, session)` validates coupon via `Coupon.is_valid()`, sets `coupon_id`, increments `coupon.used_count` atomically, stores `coupon_discount` amount
- [ ] `SaleOrder.total` computed property: `subtotal - coupon_discount + tax_total + shipping_amount`
- [ ] `SaleOrderLine.tax_amount` computed property: `unit_price * qty * tax_rate`
- [ ] `SaleOrderLine.subtotal` computed property: `unit_price * qty - discount_amount`
- [ ] All models use `TimestampMixin`

## Technical Details

**SaleOrder** (`order.py`):
- Fields: `id`, `partner_id` (FK `Partner`), `billing_address_id` (FK `Address`), `shipping_address_id` (FK `Address`), `shipping_method_id` (nullable FK `ShippingMethod`), `coupon_id` (nullable FK `Coupon`), `state` (Enum: `draft`, `confirmed`, `processing`, `shipped`, `delivered`, `cancelled`, `refunded`, default `draft`), `tracking_number`, `coupon_discount` (Numeric 12,2, default 0), `shipping_amount` (Numeric 12,2, default 0), `note`, `created_at`, `updated_at`
- Relationships: `partner`, `billing_address`, `shipping_address`, `shipping_method`, `coupon`, `lines`, `payment_transactions`
- Computed properties: `subtotal`, `tax_total`, `total`
- Methods: `confirm()`, `cancel()`, `mark_shipped(tracking_number)`, `mark_delivered()`, `apply_coupon(code, session)`

**SaleOrderLine** (`order.py`):
- Fields: `id`, `order_id` (FK `SaleOrder`), `variant_id` (FK `ProductVariant`), `qty` (Numeric 10,4), `unit_price` (Numeric 12,4), `discount_amount` (Numeric 12,2, default 0), `tax_rate` (Numeric 5,4, default 0)
- Relationships: `order`, `variant`
- Computed properties: `subtotal`, `tax_amount`

State machine wiring: use `StateMachine` from `ecommerce_core.utils.state_machine` with the transition map above.
