# Task: ShippingMethod Stub & PaymentTransaction

## Description

Implement `ShippingMethod` (a fixed-price stub with an overridable `estimate_price` hook) and `PaymentTransaction` (a full state machine for tracking payment attempts). Both are linked from `SaleOrder`.

## Acceptance Criteria
- [ ] `ecommerce_core/models/shipping.py` — `ShippingMethod` model
- [ ] `ecommerce_core/models/payment.py` — `PaymentTransaction` model
- [ ] `ShippingMethod.estimate_price(order)` raises `NotImplementedError` — agents override for dynamic pricing
- [ ] `ShippingMethod.price` (fixed Numeric) used as fallback when `estimate_price` not overridden
- [ ] `PaymentTransaction.state` state machine enforced:
  - Valid transitions: `draft→pending`, `pending→authorized`, `authorized→captured`, `pending→failed`, `authorized→failed`, `captured→refunded`
  - Invalid transitions raise `ValueError`
- [ ] `PaymentTransaction.authorize()`, `capture()`, `fail()`, `refund(amount)` method stubs — each transitions state and records `provider_ref` / `refunded_amount` as applicable
- [ ] `PaymentTransaction.refund(amount)` validates `amount <= self.amount`; stores `refunded_amount`
- [ ] `PaymentTransaction` linked to `SaleOrder` via `order_id` FK
- [ ] All models use `TimestampMixin`, `ActiveMixin` (ShippingMethod only), `CurrencyMixin` (PaymentTransaction)

## Technical Details

**ShippingMethod** (`shipping.py`):
- Fields: `id`, `name`, `carrier` (str — e.g. `'fedex'`, `'dhl'`), `price` (Numeric 10,2, fixed rate), `active`, `created_at`, `updated_at`
- Method: `estimate_price(order) -> Decimal` — raises `NotImplementedError`

**PaymentTransaction** (`payment.py`):
- Fields: `id`, `order_id` (FK `SaleOrder`), `provider` (str — e.g. `'stripe'`, `'paypal'`), `provider_ref` (str, nullable — external transaction ID), `amount` (Numeric 12,2), `currency` (str, default `'USD'`), `state` (Enum: `draft`, `pending`, `authorized`, `captured`, `failed`, `refunded`, default `draft`), `refunded_amount` (Numeric 12,2, default 0), `error_message` (Text, nullable), `created_at`, `updated_at`
- Relationships: `order`
- Methods: `authorize(provider_ref=None)`, `capture()`, `fail(error_message=None)`, `refund(amount)`

## Dependencies

Task 007 (SaleOrder) must be complete — `PaymentTransaction.order_id` FKs to `SaleOrder`.
