# Task: Partner & Address Models

## Description

Implement `Partner` (customer / company record) and `Address` (shipping / billing address linked to a partner). These are referenced by `Cart`, `SaleOrder`, and `PaymentTransaction`.

## Acceptance Criteria
- [ ] `ecommerce_core/models/partner.py` — `Partner` and `Address` models
- [ ] `Partner` has `name`, `email` (unique), `phone`, `company_name`, `is_company` flag, `active`, timestamps
- [ ] `Address` has `address_type` enum (`shipping` / `billing`), full address fields, FK to `Partner`
- [ ] `Partner.addresses` relationship returns list of `Address` records
- [ ] `Partner.default_shipping` and `Partner.default_billing` convenience properties return first address of each type or `None`
- [ ] All models use `TimestampMixin` and `ActiveMixin`

## Technical Details

**Partner** (`partner.py`):
- Fields: `id`, `name`, `email` (unique, indexed), `phone`, `company_name`, `is_company` (bool, default False), `active`, `created_at`, `updated_at`
- Relationship: `addresses`
- Properties: `default_shipping`, `default_billing`

**Address** (`partner.py`):
- Fields: `id`, `partner_id` (FK `Partner`), `address_type` (Enum: `shipping`, `billing`), `street`, `street2`, `city`, `state`, `country`, `zip_code`, `is_default` (bool), `created_at`, `updated_at`
- Relationship: `partner`

## Dependencies

Task 001 (scaffold) must be complete — needs `Base`, `TimestampMixin`, `ActiveMixin`.
