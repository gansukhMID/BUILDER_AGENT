# Task: Coupon Model

## Description

Implement the `Coupon` model with validation logic. Coupons are applied to `Cart` and transferred to `SaleOrder` at checkout. `SaleOrder.apply_coupon()` is implemented in task 007.

## Acceptance Criteria
- [ ] `ecommerce_core/models/coupon.py` — `Coupon` model
- [ ] `Coupon.code` is unique and indexed
- [ ] `Coupon.is_valid(order_amount)` returns `True` / `False` — checks expiry, usage limit, and minimum order amount
- [ ] `discount_type` supports `percentage` and `fixed` modes
- [ ] `usage_limit = None` means unlimited uses
- [ ] `Coupon.compute_discount(subtotal) -> Decimal` returns discount amount (capped at subtotal for `fixed` type)
- [ ] All fields documented with one-line docstrings
- [ ] Uses `TimestampMixin`

## Technical Details

**Coupon** (`coupon.py`):
- Fields:
  - `id`
  - `code` (str, unique, indexed)
  - `discount_type` (Enum: `percentage`, `fixed`)
  - `discount_value` (Numeric 10,4 — percent e.g. `10.00` or fixed amount e.g. `5.00`)
  - `min_order_amount` (Numeric 12,2, default 0)
  - `usage_limit` (int, nullable — `None` = unlimited)
  - `used_count` (int, default 0)
  - `expiry_date` (Date, nullable)
  - `active` (bool, default True)
  - `created_at`, `updated_at`

- Methods:
  - `is_valid(order_amount: Decimal) -> bool`
  - `compute_discount(subtotal: Decimal) -> Decimal`
