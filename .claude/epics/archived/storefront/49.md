# Task: Frontend — Product grid and product detail pages

## Description

Build the storefront homepage (product grid) and the product detail page with variant selector and "Add to Cart" functionality.

Files to create:
- `APPS/odoo-frontend/frontend/app/(store)/store/page.tsx`
- `APPS/odoo-frontend/frontend/app/(store)/store/products/[id]/page.tsx`

## Acceptance Criteria
- [ ] `/store` renders a responsive grid of product cards (name, price, "View" link)
- [ ] Product cards link to `/store/products/[id]`
- [ ] `/store/products/[id]` shows product name, list price, and a list of variants
- [ ] Each variant shows sku and price; user can select one
- [ ] "Add to Cart" button adds the selected variant to the cart (uses `useCart().addItem`)
- [ ] Adding to cart shows a brief success state (button text changes to "Added!")
- [ ] Breadcrumb: "Shop → [Product Name]"
- [ ] If product not found (API 404), show a friendly error

## Technical Details

- Fetch from `GET /store/products` and `GET /store/products/{id}` (public, no auth header needed)
- Product grid: CSS grid, 3 columns on desktop, 2 on tablet, 1 on mobile (Tailwind)
- Product detail: variant radio buttons or a simple select dropdown
- Use `lib/api.ts` (or a new store-api module) for fetching

## Dependencies

- Task 001 (backend `/store/products` endpoint)
- Task 002 (CartContext for addItem)
- Task 003 (store layout wraps these pages)

## Effort Estimate
- Size: M
- Hours: 3

## Definition of Done
- [ ] Code implemented
- [ ] TypeScript compiles with no errors
- [ ] Product grid loads and is clickable
- [ ] Add to cart updates the navbar badge
