# Task: Frontend — Cart page and checkout page

## Description

Build the cart review page and the checkout page (coupon + order placement). Checkout requires authentication.

Files to create:
- `APPS/odoo-frontend/frontend/app/(store)/store/cart/page.tsx`
- `APPS/odoo-frontend/frontend/app/(store)/store/checkout/page.tsx`

## Acceptance Criteria
- [ ] `/store/cart` lists all cart items: product name, variant sku, unit price, qty (increment/decrement), subtotal, remove button
- [ ] Cart page shows order total at the bottom
- [ ] Empty cart shows friendly message + link to `/store`
- [ ] "Proceed to Checkout" button links to `/store/checkout`
- [ ] `/store/checkout` redirects to `/store/login?next=/store/checkout` if unauthenticated
- [ ] Checkout shows cart summary (read-only line items + total)
- [ ] Coupon code field: on apply, calls `POST /store/coupons/validate`; shows discount or inline error
- [ ] Final total updates to reflect coupon discount
- [ ] "Place Order" calls `POST /store/orders` with cart lines + coupon_code
- [ ] On success: clears cart and redirects to `/store/orders/[id]`
- [ ] On API error: shows error message inline

## Technical Details

- Cart page: use `useCart()` for items, `updateQty`, `removeItem`, `totalPrice`
- Checkout page: read cart from `useCart()` — if empty, redirect to `/store/cart`
- Coupon state: local `useState` for code input, discount result
- Auth check on checkout: read `access_token` cookie; if absent, redirect
- Order placement: `POST /store/orders` body: `{ lines: items.map(i => ({ variant_id: i.variantId, qty: i.qty })), coupon_code }`
- After successful order, call `clearCart()` before redirecting

## Dependencies

- Task 001 (backend `/store/coupons/validate` and `/store/orders` endpoints)
- Task 002 (CartContext)
- Task 003 (store layout, auth state)
