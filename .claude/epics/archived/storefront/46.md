# Task: Backend — /store router, schemas, register in main.py

## Description

Create the FastAPI `/store` router and its Pydantic schemas, then register it in `main.py`. This is the foundation for all storefront frontend work.

Files to create/edit:
- `APPS/odoo-frontend/backend/routers/store.py` (new)
- `APPS/odoo-frontend/backend/schemas/store.py` (new)
- `APPS/odoo-frontend/backend/main.py` (edit — add `from routers import store` and register router)

## Acceptance Criteria
- [ ] `GET /store/products` returns 200 with product list (no auth required)
- [ ] `GET /store/products/{id}` returns product + variants (no auth)
- [ ] `POST /store/coupons/validate` returns `{valid, discount_type, discount_value}` or 404 (no auth)
- [ ] `POST /store/orders` returns 401 without JWT, 201 with valid JWT + cart lines
- [ ] `GET /store/orders` returns current user's orders only (auth required)
- [ ] `GET /store/orders/{id}` returns 403 when accessing another user's order
- [ ] All endpoints appear in FastAPI `/docs`

## Technical Details

### schemas/store.py
```python
class StoreVariantOut(BaseModel):
    id: int
    sku: Optional[str]
    default_price: float

class StoreProductOut(BaseModel):
    id: int
    name: str
    list_price: float
    variant_count: int = 0

class StoreProductDetailOut(BaseModel):
    id: int
    name: str
    list_price: float
    variants: List[StoreVariantOut]

class CouponValidateRequest(BaseModel):
    code: str

class CouponValidateOut(BaseModel):
    valid: bool
    discount_type: str
    discount_value: float

class CartLineIn(BaseModel):
    variant_id: int
    qty: float

class CreateOrderRequest(BaseModel):
    lines: List[CartLineIn]
    coupon_code: Optional[str] = None

class StoreOrderLineOut(BaseModel):
    id: int
    variant_id: Optional[int]
    qty: float
    unit_price: float
    subtotal: float

class StoreOrderOut(BaseModel):
    id: int
    reference: str
    state: str
    total_amount: float

class StoreOrderDetailOut(BaseModel):
    id: int
    reference: str
    state: str
    total_amount: float
    coupon_discount: float
    lines: List[StoreOrderLineOut]
```

### routers/store.py
- Public endpoints: list products, get product — same logic as ecommerce router but no auth
- `POST /store/coupons/validate`: query Coupon by code, return discount info or raise 404
- `POST /store/orders`: 
  - Require `get_current_user`
  - For each CartLineIn, fetch ProductVariant by id, use `default_price` as unit_price
  - Create `SaleOrder` (set `user_id` on the order — note SaleOrder may not have user_id; store user_id via a note or just create the order)
  - Create `SaleOrderLine` per cart item
  - If coupon_code provided, validate and apply discount to order total
  - Commit and return order
- `GET /store/orders`: filter SaleOrders where they belong to current user
- `GET /store/orders/{id}`: get SaleOrder, verify ownership, return detail

Note: `SaleOrder` from `ecommerce_core` may not have a `user_id` column. In that case, store the mapping using the existing `user_core.models.order.AbstractOrder` (the concrete `Order` model in `models/concrete.py` already has `user_id`). Use `Order` (user_core) for user-linked orders, not `SaleOrder` (ecommerce_core). Alternatively, create a `UserOrder` join table or just filter by checking if there's a way to link. 

IMPORTANT: Check the actual model fields before writing code. Read:
- `TEMPLATE_CORE/ECOMMERCE/ecommerce_core/models/order.py`
- `TEMPLATE_CORE/ECOMMERCE/ecommerce_core/models/coupon.py`
- `TEMPLATE_CORE/USER/user_core/models/order.py`
- `APPS/odoo-frontend/backend/models/concrete.py`

Use whatever pattern makes the data link work correctly.
