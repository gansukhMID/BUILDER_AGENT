# Task: Frontend — Store layout, navbar, auth pages

## Description

Create the `(store)` route group layout with a consumer-style navbar, and the login/register pages for the storefront. No admin sidebar involved.

Files to create:
- `APPS/odoo-frontend/frontend/app/(store)/layout.tsx`
- `APPS/odoo-frontend/frontend/app/(store)/store/login/page.tsx`
- `APPS/odoo-frontend/frontend/app/(store)/store/register/page.tsx`

## Acceptance Criteria
- [ ] `(store)/layout.tsx` wraps all store pages with `CartContext.Provider`
- [ ] Navbar shows: brand name ("Shop"), "Products" link → `/store`, cart icon with item count badge, auth state
- [ ] Auth state: "Login" link when unauthenticated; user email + "Logout" button when authenticated
- [ ] Logout clears `access_token` cookie and redirects to `/store`
- [ ] `/store/login` has email + password form; on success sets cookie and redirects to intended page or `/store`
- [ ] `/store/register` has email + password form; on success sets cookie and redirects to `/store`
- [ ] Both auth forms show inline error messages on failure
- [ ] Store layout is visually distinct from admin (light background, no sidebar)

## Technical Details

- Auth state: read `access_token` cookie client-side. Call `GET /auth/me` on mount to get user info.
- After login/register: set cookie `access_token=<token>; path=/; max-age=86400`
- Cart badge: use `useCart().totalItems`
- The navbar is a client component (`'use client'`)
- Layout wraps children with `<CartProvider>`

## Dependencies

- Task 002 (CartContext) — layout imports CartProvider
