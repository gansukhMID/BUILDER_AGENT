# Task: Frontend — Order history pages

## Description

Build the customer's order list and order detail pages. Both require authentication and show only the current user's orders.

Files to create:
- `APPS/odoo-frontend/frontend/app/(store)/store/orders/page.tsx`
- `APPS/odoo-frontend/frontend/app/(store)/store/orders/[id]/page.tsx`

## Acceptance Criteria
- [ ] `/store/orders` redirects to `/store/login?next=/store/orders` if unauthenticated
- [ ] `/store/orders` lists orders: reference (SO-N), state badge, total amount, date, "View" link
- [ ] Empty state: "No orders yet" + link to `/store`
- [ ] `/store/orders/[id]` shows order header (reference, state, total) and line items (variant sku, qty, unit price, subtotal)
- [ ] Shows coupon discount row if discount > 0
- [ ] Breadcrumb: "Orders → SO-N"
- [ ] If order not found or forbidden (403), show error message

## Technical Details

- Fetch from `GET /store/orders` and `GET /store/orders/{id}` (auth required — send Bearer token from cookie)
- State badge: colored pill matching order state (draft=gray, confirmed=blue, processing=yellow, done=green, cancelled=red)
- Auth check: read `access_token` cookie; redirect if absent
- Use `lib/api.ts` with Authorization header

## Dependencies

- Task 001 (backend `/store/orders` endpoints)
- Task 003 (store layout, auth state)
