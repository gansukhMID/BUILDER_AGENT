# Task: Location Model + Hierarchy Utilities

## Description

Implement the `Location` model — a self-referential hierarchical storage node — and location tree utility functions.

Files to create:
- TEMPLATE_CORE/WMS/wms_core/models/location.py
- TEMPLATE_CORE/WMS/wms_core/utils/hierarchy.py

Location model fields:
- `id: Mapped[int]` (primary key)
- `parent_id: Mapped[Optional[int]]` (FK to self, nullable)
- `location_type: Mapped[str]` (enum: internal, view, customer, supplier, transit, inventory, production)
- Inherits: `TimestampMixin`, `ActiveMixin`, `NameMixin`
- Relationship: `parent: Mapped[Optional["Location"]]`, `children: Mapped[list["Location"]]`
- Property: `complete_name` — walks parent chain to build "Parent/Child/Grandchild" path
- Method: `is_ancestor_of(other: "Location") -> bool`

utils/hierarchy.py functions:
- `build_full_path(location: Location) -> str` — same as complete_name but standalone
- `get_all_children(location_id: int, session: Session) -> list[Location]` — recursive CTE or loop

## Acceptance Criteria
- [ ] `from wms_core.models.location import Location` succeeds
- [ ] `Location.__tablename__` = `"stock_location"`
- [ ] A 3-level hierarchy `WH → Stock → Shelf-A` gives `location.complete_name == "WH/Stock/Shelf-A"`
- [ ] `is_ancestor_of` returns True for parent, False for sibling
- [ ] `get_all_children` returns all descendants (not just direct children)
- [ ] Self-referential FK uses `back_populates` correctly

## Technical Details

Use SQLAlchemy `adjacency_list` pattern for self-referential. `complete_name` should be a Python `@property` that walks `self.parent` chain (not a DB-computed column) — keep it simple. Location type stored as `String` with application-level enum validation.
