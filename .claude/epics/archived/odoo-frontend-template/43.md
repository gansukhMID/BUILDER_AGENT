# Task: WMS Dashboard Pages

## Description

Implement all WMS frontend pages using the API from issue 005 and the shell from issue 008.

Pages:
- `/wms` — Warehouse list: table with name, code, location count
- `/wms/locations` — Location tree: indented list showing complete_name hierarchy
- `/wms/inventory` — Inventory table: product, location, lot, quantity; filterable by location
- `/wms/pickings` — Picking list: table with reference, state badge, picking type
- `/wms/pickings/[id]` — Picking detail: header info + moves table

## Acceptance Criteria
- [ ] All 5 WMS pages render without console errors when backend is running with seed data
- [ ] Warehouse list shows at least name and code columns
- [ ] Location tree indented by hierarchy depth (use `complete_name` split on `/` for depth)
- [ ] Inventory table has location filter (dropdown populated from /wms/locations)
- [ ] Picking list shows state with color-coded Badge (draft=gray, confirmed=blue, in_progress=yellow, done=green)
- [ ] Picking detail shows moves table with product name, qty_demand, qty_done
- [ ] Loading spinner shown during data fetch
- [ ] Empty state message when no data

## Technical Details

All pages are Server Components (async) using `lib/api.ts`. State badges use the `Badge` component from components/ui/. DataTable component used for all tabular data. Location tree: sort by complete_name and indent by counting `/` separators.
