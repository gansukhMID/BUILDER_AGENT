# Task: ECOMMERCE API Routers + /stats

## Description

Implement `routers/ecommerce.py` with 5 endpoints and `schemas/ecommerce.py`. Also add `GET /stats` endpoint (can live in a separate `routers/stats.py` or in ecommerce router). Endpoints: GET /ecommerce/products, GET /ecommerce/products/{id}, GET /ecommerce/orders, GET /ecommerce/orders/{id}, GET /ecommerce/coupons, GET /stats.

## Acceptance Criteria
- [ ] `GET /ecommerce/products` returns list of `{id, name, list_price, variant_count}`
- [ ] `GET /ecommerce/products/{id}` returns template detail + variants list `{sku, default_price, attribute_summary}`
- [ ] `GET /ecommerce/orders` returns list of `{id, reference, state, total_amount, partner_name}`
- [ ] `GET /ecommerce/orders/{id}` returns order + lines `{product_name, quantity, unit_price, line_total}` + payment_transaction state
- [ ] `GET /ecommerce/coupons` returns list of `{code, discount_type, discount_value, usage_count, usage_limit}`
- [ ] `GET /stats` returns `{users, active_memberships, warehouses, open_pickings, products, open_orders}`
- [ ] All Pydantic v2 schemas with `from_attributes=True`

## Technical Details

Import ProductTemplate, ProductVariant, SaleOrder, SaleOrderLine, Coupon, PaymentTransaction from ecommerce_core.models. For /stats, query each table for counts. variant_count = subquery count of ProductVariant where template_id = id.
