# Task: WMS API Routers

## Description

Implement `routers/wms.py` with 6 endpoints and `schemas/wms.py` with Pydantic response models. Endpoints: GET /wms/warehouses, GET /wms/locations (with complete_name), GET /wms/products, GET /wms/inventory (quants with product+location names), GET /wms/pickings (list), GET /wms/pickings/{id} (detail with moves).

## Acceptance Criteria
- [ ] `GET /wms/warehouses` returns list of `{id, name, code}`
- [ ] `GET /wms/locations` returns list of `{id, name, complete_name, location_type}` ordered by complete_name
- [ ] `GET /wms/products` returns list of `{id, name, tracking_type, active}`
- [ ] `GET /wms/inventory` returns list of `{product_name, location_name, lot_name, quantity}`
- [ ] `GET /wms/pickings` returns list of `{id, name, state, picking_type_name}`
- [ ] `GET /wms/pickings/{id}` returns picking + list of moves `{product_name, qty_demand, qty_done}`
- [ ] All schemas use Pydantic v2 `model_config = ConfigDict(from_attributes=True)`

## Technical Details

Import Warehouse, Location, Product, Quant, Picking, Move from wms_core.models. Use SQLAlchemy session from db.py. No auth required on WMS read endpoints in this task (add later if needed).
