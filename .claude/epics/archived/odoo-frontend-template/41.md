# Task: USER API Routers

## Description

Implement `routers/users.py` with 4 endpoints and `schemas/users.py`. Endpoints: GET /users, GET /users/{id}, GET /users/{id}/activity, GET /users/{id}/orders.

## Acceptance Criteria
- [ ] `GET /users` returns list of `{id, email, membership_tier, last_login_at, is_active}`
- [ ] `GET /users/{id}` returns user + profile `{display_name, avatar_url, bio, phone, timezone, locale}` + current membership `{tier, started_at, expires_at}`
- [ ] `GET /users/{id}/activity` returns last 50 ActivityEvent records `{event_type, occurred_at, ip_address, metadata}`
- [ ] `GET /users/{id}/orders` returns list of Order records for that user `{reference, state, total_amount, created_at}`
- [ ] Returns 404 if user not found

## Technical Details

Import concrete User, UserProfile, Membership, ActivityEvent, Order from models/concrete.py (the subclasses created in issue 002). Use SQLAlchemy joined loads for profile/membership. membership_tier: query latest active Membership for user.
