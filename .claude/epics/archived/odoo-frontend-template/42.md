# Task: Next.js Frontend Scaffold + Odoo Shell Layout

## Description

Initialize Next.js 15 App Router project in `APPS/odoo-frontend/frontend/`. Create the Odoo-style shell: collapsible sidebar with module nav (WMS / Ecommerce / Users), top navbar with user email + logout button, breadcrumb component. Create login page (`/login`) with email+password form + JWT storage in cookie. Create middleware.ts for route protection. Create `lib/api.ts` typed fetch wrapper.

## Acceptance Criteria
- [ ] `npm run dev` starts without errors, root redirects to `/login` if unauthenticated
- [ ] `/login` page has email + password form; on submit calls `POST /auth/login`, stores token in cookie, redirects to `/dashboard`
- [ ] All authenticated routes wrapped in OdooShell layout with sidebar + navbar
- [ ] Sidebar shows: Dashboard, WMS (with sub-items: Warehouses, Locations, Inventory, Pickings), Ecommerce (Products, Orders, Coupons), Users
- [ ] Active sidebar item highlighted
- [ ] `lib/api.ts` exports `get<T>(path)`, `post<T>(path, body)` with auto JWT injection from cookie and base URL from `NEXT_PUBLIC_API_URL`
- [ ] `middleware.ts` redirects unauthenticated requests to `/login` (except `/login` itself)
- [ ] TypeScript strict mode, no type errors (`npm run build` passes)

## Technical Details

Use Next.js App Router with `app/layout.tsx` as root. Create `app/(auth)/login/page.tsx` (no shell) and `app/(app)/layout.tsx` (with OdooShell). Sidebar: fixed left panel, 240px wide, dark background (#1f2937), white text. Navbar: top bar with app title + user info. Use Tailwind for all styling — no component library.
