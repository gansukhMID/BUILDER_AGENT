# Task: Auth API (register, login, JWT)

## Description

Implement full auth flow in `routers/auth.py` and `schemas/auth.py`. Register creates a User (concrete model from 002) with bcrypt password hash. Login validates hash, returns JWT. /auth/me returns current user from token. Create `auth/utils.py` with `hash_password()`, `verify_password()`, `create_access_token()`, `get_current_user()` dependency.

## Acceptance Criteria
- [ ] `POST /auth/register` with `{"email":"x@x.com","password":"pass"}` creates user, returns `{"access_token": "...", "token_type": "bearer"}`
- [ ] `POST /auth/login` with valid credentials returns JWT; invalid credentials return 401
- [ ] `GET /auth/me` with Bearer token returns `{"id": 1, "email": "...", "is_active": true}`
- [ ] Passwords stored as bcrypt hash (not plaintext) in DB
- [ ] JWT expires in 24 hours

## Technical Details

Use `python-jose` for JWT (HS256, SECRET_KEY from env). Use `passlib[bcrypt]` for password hashing. AbstractUser.set_password() and check_password() are stubs — override them in concrete User model or implement directly in utils.py.

## Dependencies

- 003 (FastAPI Foundation) — main.py with router includes must exist; can run in parallel with 005.
